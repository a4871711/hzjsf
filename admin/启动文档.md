# 矢历运动后台管理系统 — 启动文档

> 前端：Vue 2 + Element UI（renren-fast-vue 定制），webpack-dev-server 跑在 **8001** 端口
> 后端：renren-security（Java），跑在 **8080** 端口
> 前端开发时通过 `/proxyApi` 代理到后端，**后端不起，前端登录/接口全部失败**。

---

## 0. 当前状态（截至本文档生成时实测）

| 检查项 | 结果 |
|---|---|
| 后端 8080 | ✅ 在跑（java 进程），`curl .../captcha.jpg` 返回 HTTP 200 |
| 前端 8001 | ✅ 已在跑（webpack-dev-server），`curl http://127.0.0.1:8001` 返回 HTTP 200 |
| 依赖 node_modules | ✅ 已安装 |

**所以现在直接打开浏览器访问即可：** http://localhost:8001

下面是"从零启动 / 重启"时的标准步骤。

---

## 1. 环境要求

- **Node.js 16**（项目锁定，见 `.nvmrc`；本机已装 `v16.20.2`）
  - ⚠️ 本机默认 node 是 v25，**老的 webpack 3 在 node 25 下容易报错**，务必切到 16。
- 后端服务（renren-security）需要先在 `127.0.0.1:8080` 起好。

---

## 2. 启动步骤

### 第 1 步：确认后端在跑
```bash
curl -s -o /dev/null -w "%{http_code}\n" http://127.0.0.1:8080/renren-security/captcha.jpg
# 返回 200 = 后端 OK；连接失败 = 要先去启动后端 Java 服务
```

### 第 2 步：切到 Node 16
```bash
cd "/Users/macmima0000/Desktop/jianshen/健身房代码/gym/admin"
nvm use 16          # 或 nvm use（自动读 .nvmrc）
node -v             # 确认显示 v16.x
```

### 第 3 步：安装依赖（首次 / node_modules 缺失时才需要）
```bash
npm install
```
> 本机已装好，平时不用重复跑。

### 第 4 步：启动前端开发服务器
```bash
npm run dev
```
- 端口 **8001**，会自动打开浏览器。
- 若提示 8001 被占用，说明已经有一个在跑了——直接访问 http://localhost:8001 即可，不用再起。

---

## 3. 访问与登录

- 地址：**http://localhost:8001**
- 登录页需要 **用户名 + 密码 + 验证码**（验证码图片由后端 `captcha.jpg` 提供）。
- 账号密码由后端数据库决定（renren 系默认常为 `admin/admin`，本定制版以实际后端为准）。

---

## 4. 关键配置位置（要改时看这里）

| 想改什么 | 文件 |
|---|---|
| 后端代理地址（dev） | `config/index.js` → `proxyTable['/proxyApi'].target` |
| 接口域名 / 图片域名 | `.env`（dev）、`.env.production`（prod）→ `VUE_APP_URL` / `VUE_APP_IMG` |
| 前端端口 | `config/index.js` → `dev.port`（默认 8001） |

---

## 6. 常见问题排查

- **页面能开但登录/接口报错** → 后端 8080 没起，或 `config/index.js` 代理地址不对。
- **`npm run dev` 报 node-sass / webpack 相关错误** → 没切到 Node 16，先 `nvm use 16`。
- **菜单页面空白 / 路由 404** → 本项目路由是**动态菜单驱动**：菜单从后端 `/sys/menu/nav` 拉取，后端没有对应菜单记录时前端不会注册该路由。新加的页面要先在后端"菜单管理"里配好对应 url。
