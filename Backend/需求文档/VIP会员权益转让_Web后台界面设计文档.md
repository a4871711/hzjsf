# VIP 会员权益转让 — Web 后台界面设计文档（给 Claude design 画图用）

> 配套文档：《VIP会员权益转让_需求梳理与实现方案.md》（业务规则与决策点以那份为准）
> 适用范围：**仅 Web 后台管理端**（`admin/` 项目，给门店/运营用），不含移动端小程序。
> 风格基准：现有后台 `renren-fast-vue`（Vue 2.5 + **Element UI 2.13**），矢历运动健身房后台。**所有新页面必须和现有页面长一个样**。

---

## 0. 给 Claude design 的总体指引（先读这段）

- **画什么**：下面第 4~9 节每一节 = 一张完整的后台界面（含左侧边栏 + 顶栏 + 内容区），按桌面 Web 出图。
- **画布**：桌面 Web，宽 **1440px**（内容区按 1440 减去侧栏宽算）。中文界面。
- **视觉语言**：标准 **Element UI 2.x（chalk 主题）** 外观——蓝色主操作、浅灰内容底、白色卡片、4px 圆角、表格斑马纹、`mini` 尺寸操作按钮。**不要**自创花哨风格，越"像后台管理系统模板"越对。
- **配色/间距/组件**：严格用第 1、3 节的设计令牌与组件规范。
- **每页结构固定**：顶部「搜索栏」→ 中部「数据表格 + 分页」→ 浮层「新增/编辑弹窗」。这是现有所有页面的统一范式，照搬。

## 3. 通用组件规范（所有页面共用）

**① 搜索栏（页面顶部白条）**：一行行内排布——若干输入框/下拉（各宽约 200px）+ 蓝色「搜索」主按钮 +（可选）「重置」。右侧或下一行放「+ 添加」主按钮。

**② 数据表格**：Element `el-table`，带表头底色 `#FAFAFA`、斑马纹、悬停高亮。
- 「状态」列用 **el-switch**（开=上架/启用，绿色）或彩色 **el-tag**。
- 最右「操作」列固定，放 `mini` 按钮：编辑（绿 success + ✎图标）、删除（红 danger + 🗑图标），按需加「审核/详情」。
- 金额右对齐、加「¥」；时间列 `YYYY-MM-DD HH:mm:ss`。

**③ 分页**：表格右下角 Element `el-pagination`（总数 + 每页条数 + 页码），激活页蓝色。

**④ 新增/编辑弹窗**：Element `el-dialog`，宽 **700px**，标题「新增/编辑 XXX」。内部 `el-form` 左对齐标签、`label-width:150px`，控件宽约 350px。底部右对齐「确认」（蓝主按钮 + ⊕图标）「取消」。

**⑤ 状态标签配色**（转让状态机，统一这套）：
| 状态 | 颜色 |
|---|---|
| 待付费 / 待审核 | 橙 `#E6A23C`（warning tag） |
| 待受让人确认 | 蓝 `#409EFF`（primary tag） |
| 已生效 | 绿 `#67C23A`（success tag） |
| 已驳回 / 已超时 / 已撤回 | 灰 `#909399`（info tag）；驳回可红 |

**⑥ 其它**：删除/驳回前弹「确定要…吗?」确认框；操作成功右上角绿色 toast；加载时表格区遮罩 loading；无数据显示「暂无数据」空态；无权限的按钮直接不显示。

## 5. 页面②：转让费用规则配置（`sys/vipFeeRule`）

> 实现 D-4：转让费用「按转让次数分档」。一条规则 = 一张"第几次→收多少"的阶梯表，权益卡关联某条规则。

```
┌─ 内容区卡片 ───────────────────────────────────────────────────────────┐
│ [规则名称 ____]   [🔍 搜索]                                  [ + 添加 ] │
├────────────────────────────────────────────────────────────────────────┤
│ ID │ 规则名称      │ 分档摘要                        │ 状态 │ 操作       │
│ 1  │ 标准转让费用  │ 第1次免费/第2次起¥50/第3次起¥100│ 启用 │ ✎编辑 🗑删除│
└────────────────────────────────────────────────────────────────────────┘
```

**新增/编辑弹窗（标题"新增/编辑 费用规则"）**：
- 规则名称（input，必填）
- **分档费用表**（可增删行的小表格，Element 风格内嵌表格）：

```
┌─────────────────────────────────────────────┐
│ 从第几次转让起 │ 收费（元） │ 操作            │
│   第 1 次      │    0       │ 删除            │
│   第 2 次      │   50       │ 删除            │
│   第 3 次      │  100       │ 删除            │
│              [ + 添加档位 ]                    │
└─────────────────────────────────────────────┘
说明文字：按该权益"已被转让的次数"命中对应档位；最后一档兜底所有更高次数。第1次可设为0=免费。
```
- 底部「确认 / 取消」。

```
┌─ 内容区卡片 ───────────────────────────────────────────────────────────────────┐
│ [状态 ▾] [转让人手机 ___] [受让人手机 ___] [发起时间 起 ~ 止]   [🔍 搜索]       │
├───────────────────────────────────────────────────────────────────────────────┤

│T2026..│年度VIP│张三 138…│李四 139…│¥50│旗舰店→社区店│待审核│2026-…│审核 详情│
│T2026..│季度VIP│王五 136…│赵六 137…│免费│旗舰店→旗舰店│已生效│2026-…│详情      │

│                                                  共 8 条  < 1 >  10条/页 ▾        │
└───────────────────────────────────────────────────────────────────────────────┘
```

**搜索栏**：状态（select：待审核/已通过/待受让人确认/已生效/已驳回/已超时/已撤回）、转让人手机、受让人手机、发起时间范围、搜索。

**表格列**：转让单号、权益卡名称、转让人(昵称+手机)、受让人(昵称+手机)、转让费用(¥/免费)、门店(转让人门店→受让人门店)、状态(彩色 tag 见 3⑤)、发起时间、操作（「审核」仅待审核时显示，蓝主按钮；「详情」始终有）。

**审核弹窗（点"审核"，标题"转让审核"，宽 720）**：
```
┌─ 转让审核 ─────────────────────────────────────────┐
│ 转让单号：T2026...        发起时间：2026-...        │
│ 权益卡：年度VIP权益   剩余有效期：到 2026-12-31     │
│ 转让人：张三 138****1234   门店：旗舰店             │
│ 受让人：李四 139****5678   门店：社区店             │
│ 转让费用：¥50（转让人已支付）                       │
│ ─────────── 资质校验（系统自动核验）───────────── │
│   权益是否在有效期 …………………… ✅ 通过            │
│   转让人账号状态(未封禁/黑名单) … ✅ 通过            │
│   是否欠费 ……………………………… ✅ 无              │
│   是否办理退卡 …………………………… ✅ 无              │
│   重大违规记录 ………………………… ✅ 无              │
│   适用门店校验:受让人门店【社区店】∈ 权益适用门店 … ✅ 通过 │
│ ─────────────────────────────────────────────── │
│ 驳回理由：[________________________]（驳回时必填）  │
│                          [ 驳回(红) ]  [ 通过(蓝) ] │
└────────────────────────────────────────────────────┘
```
- 校验项命中异常时显示 ❌ 红色 + 原因；任一异常时「通过」按钮置灰，引导驳回。
- 「通过」→ 二次确认 → 状态转「待受让人确认」并推送受让人。
- 「驳回」→ 需填理由 → 状态转「已驳回」+ 自动退转让费用 + 通知双方。

**详情弹窗**：同上但只读、无操作按钮，额外展示当前状态、各节点时间线（发起→审核→受让人确认→生效）。

## 10. 文案与状态约定（画图时统一用）

- **转让状态机文案**（与状态 tag 颜色对应，见 3⑤）：待付费 → 待审核 → 待受让人确认 → 已生效；分支：已驳回 / 已超时 / 已撤回。
- **空态**：表格无数据居中显示「暂无数据」+ 浅灰插画/图标。
- **金额**：一律「¥」前缀、右对齐；免费显示「免费」绿色或灰色文字。
- **时间**：`YYYY-MM-DD HH:mm:ss`。
- **按钮语义色**：主操作=蓝、编辑/通过=绿、删除/驳回/拉黑=红、次要=灰。
- **整体气质**：干净、信息密度高的企业级后台，和现有 renren 后台无缝一致；不要圆润大留白的 C 端风格。
